# O’PDCA Reference Intake Notes

This file defines intake rules for future reference assets in the O’PDCA Skill Pack.

Intake notes are **governance scaffolds only** and do **not** create runtime behavior.

## Source ID naming

Source IDs should use the `OPDCA-` prefix followed by a descriptive slug.
Examples: `OPDCA-COURSE-OVERVIEW-001`, `OPDCA-ANNUAL-BUDGET-001`.

## Reference area selection

Each source should map to **one primary reference area** from the ten defined areas.
A source may be associated with additional areas in its notes, but the primary area
must be unambiguous.

## Citation note

Citation notes should preserve the following when available:

- Source title
- Owner / author
- Date or version

## Version note

Version notes should record one of the following statuses when applicable:

- `draft`
- `reviewed`
- `accepted`
- `deprecated`
- `superseded`

## Review status

Review status should remain `placeholder` or `draft` until official review is complete.
Status transitions must be recorded in the source-map.md entry.

## General rules

- Intake notes are governance scaffolds only and do not create runtime behavior.
- Review status should remain placeholder or draft until official review is complete.
