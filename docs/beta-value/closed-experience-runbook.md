# S'FOCUS 闭门体验主持流程手册

## 主持人开场说明

今天这场不是正式服务，也不是普通软件测试。我们想邀请你带一个真实经营问题，和 Eliy 走一轮 S'FOCUS，看它是否能帮助你把问题说清楚、找到一个当前候选瓶颈，并形成一张小到可以执行的行动卡。

Eliy 不会替你做最终判断。它的目标是帮助你看清系统、整理判断、缩小下一步行动。最终选择权仍然在你这里。

## Eliy 当前阶段说明

Eliy 目前处在闭门体验阶段。它已经可以通过网页与用户进行 S'FOCUS 协作，但还不是成熟商业服务。

这次我们更关心它是否有价值，而不是它是否已经完美。

## 本次体验边界

本次体验只围绕 S'FOCUS 场景：

- 定义系统；
- 梳理多个问题；
- 识别当前候选瓶颈；
- 理解控制投料；
- 形成最小行动卡；
- 判断是否愿意进入后续复盘。

请不要引导用户上传公司机密、客户隐私、敏感财务数据或个人隐私。

## 老学员进入方式

1. 打开闭门体验入口；
2. 输入第一层浏览器访问账号；
3. 进入页面后输入 Eliy 内测登录账号；
4. 进入聊天页面；
5. 点击或输入 S'FOCUS 相关任务，开始体验。

主持人只协助进入，不替用户选择问题，不替用户评价行动卡。

## 建议测试任务

请用户带入一个真实但可讨论的问题，例如：

- 课程交付卡住；
- 老师时间不足；
- 学员复盘跟不上；
- 销售转化低；
- 团队执行分散；
- 多个问题同时出现，不知道先处理哪个。

建议提示语：

```text
我想用 S'FOCUS 澄清一个经营问题。现在问题很多：____。请先帮我定义系统，再帮我找当前候选瓶颈。
```

## 体验时间安排

1. 5 分钟：说明 Eliy 与 S'FOCUS 场景；
2. 5 分钟：技术通行检查；
3. 20-30 分钟：用户带入真实问题与 Eliy 互动；
4. 10 分钟：填写价值反馈；
5. 5 分钟：选择是否进入后续行动复盘。

## 技术问题处理方式

如果用户打不开页面、无法登录、无法发送消息或没有回复，先记录账号编号、设备、浏览器、出现位置和截图。

技术问题只做最小处理，不把整场体验变成软件排错。若 5 分钟内无法恢复，记录为技术未通过，并另约时间补测。

## 价值反馈收集方式

体验结束后，请用户填写：

1. S'FOCUS 价值反馈表；
2. 主体型体验观察表；
3. 后续行动复盘报名表。

主持人不要诱导用户给好评。可以提醒用户：具体说出哪句话有帮助、哪句话空泛，比打高分更重要。

## 结束后的行动卡回收方式

如果形成了最小行动卡，请用户任选一种方式保存：

- 复制行动卡文字；
- 截图；
- 由主持人按账号编号记录标题和关键字段。

重点记录：行动卡是否足够小、是否能现实执行、用户是否愿意真的执行。

## 7 天后复盘邀请方式

结束前询问：

1. 你愿意在 7 天内执行这张行动卡吗？
2. 你愿意 7 天后回来复盘一次吗？
3. 你愿意带一个新的真实问题继续体验吗？

不要把复盘包装成销售邀约。它是验证 Eliy 是否能从“说清楚问题”走到“推动行动”的下一步。
