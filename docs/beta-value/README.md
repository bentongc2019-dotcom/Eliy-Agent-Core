# S'FOCUS 闭门体验价值反馈材料

## 本次闭门体验目标

本次闭门体验不是普通软件内测，而是验证 Eliy 在 S'FOCUS 场景中是否真的能帮助老学员完成一次有价值的经营问题澄清。

重点观察 Eliy 是否能帮助用户：

1. 说清楚正在分析的系统；
2. 从多个问题中识别一个当前候选瓶颈；
3. 理解控制投料，不急于同时处理所有问题；
4. 形成一张足够小、可以执行的最小行动卡；
5. 保留用户自己的判断权；
6. 感受到 Eliy 不是普通 AI 问答，而是主体型商业智能体 / S'FOCUS 陪练。

## 本次体验对象

本轮对象是 6 月 11 日前小范围老学员闭门体验用户。建议优先邀请熟悉 S'FOCUS 或有真实经营问题需要澄清的学员。

## 本次体验不是什么

- 不是正式上线服务；
- 不是普通软件找 bug；
- 不是技术压力测试；
- 不是让用户评价界面好不好看；
- 不是让 Eliy 替用户做最终决策；
- 不是收集公司机密、客户隐私或敏感财务数据。

## 本次要验证的核心假设

1. 老学员能带入真实问题，并愿意用 Eliy 走一轮 S'FOCUS。
2. Eliy 能帮助用户把“很多问题”转成“一个当前候选瓶颈”。
3. 用户能理解“先优化瓶颈，而不是马上增加资源”。
4. 最小行动卡能让用户愿意真实执行，而不是停留在口头认可。
5. 用户能感受到 Eliy 的价值来自陪练式判断，而不是普通问答。

## 技术通行检查与价值反馈的区别

技术通行检查只确认用户能打开、登录、发送消息、看到回复和行动卡。它不是本轮主反馈表。

价值反馈关注的是：Eliy 是否帮助用户完成 S'FOCUS 的关键动作，是否形成更清楚的判断，是否愿意执行行动卡。

## 体验流程摘要

1. 5 分钟：说明 Eliy 与 S'FOCUS 场景；
2. 5 分钟：完成技术通行检查；
3. 20-30 分钟：用户带入真实问题与 Eliy 互动；
4. 10 分钟：填写价值反馈；
5. 5 分钟：选择是否进入后续行动复盘。

## 反馈回收方式

建议按账号编号回收，不要求用户填写过多个人信息。行动卡可通过复制文字、截图或主持人记录方式保存。

每位用户至少回收：

1. 技术通行检查结果；
2. S'FOCUS 价值反馈；
3. 主体型体验观察；
4. 是否愿意执行行动卡与参加 7 天复盘。

## 结果整理方式

体验结束后使用 `post-session-synthesis-template.md` 汇总，按技术问题、体验问题、价值问题、方法论问题和主体性问题分开判断。

不要把“满意度高”直接等同于价值成立。更重要的信号是：用户是否形成行动卡、是否愿意执行、是否愿意 7 天后回来复盘。
