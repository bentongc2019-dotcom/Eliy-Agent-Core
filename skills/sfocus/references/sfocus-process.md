# S’FOCUS Process Reference

## Step 0｜System
Identify the system before identifying the bottleneck.
Minimum questions:
1. What system are we discussing?
2. What is the system boundary?
3. What is the system objective?
4. What are the key elements?
5. How do these elements connect?

## Step 1｜Find
Find the bottleneck.
The bottleneck is not a general problem.
The bottleneck is the scarce resource or capability that limits system throughput.
When the user lists many problems, classify them as:
- undesirable eﬀects
- symptoms
- candidate causes
- candidate bottleneck

The assistant should keep the bottleneck as candidate until there is enough evidence.

## Step 2｜Optimize
Optimize the current bottleneck before expanding it.
The assistant should first help the user use the existing bottleneck more eﬀectively.
Examples:
- protect bottleneck time
- reduce non-essential work
- improve decision quality
- remove interruptions
- clarify input quality
- reduce rework

## Step 3｜Cooperation
Help the rest of the system cooperate with the bottleneck.
The assistant should look for starvation and overload.
Starvation means the bottleneck does not receive the right input, information, or support.
Overload means the bottleneck receives too much work, noise, or poorly prepared input.

## Step 4｜Upgrade
Upgrade only after optimization and cooperation have been attempted.
Upgrade may include:
- adding capacity
- improving tools
- changing roles
- adding people
- redesigning the system

## Step 5｜Start again
After the bottleneck is upgraded, the bottleneck may move elsewhere.
Return to Step 0 and re-identify the system.
